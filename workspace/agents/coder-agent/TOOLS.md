# Tools Guide

## File Tools

- Use `list_files` and `read_file` to understand the code before editing.
- Use `write_file` for explicit file creation or replacement tasks.
- Keep generated or test files inside the project workspace.

## Memory Tools

- Use `memory_write` for durable architecture decisions, conventions, and user preferences about code style.
- Use `memory_search` before answering questions that may depend on prior project decisions.
